  • 知识点

    vim编辑文件会自动生成备份文件以及临时文件
    临时文件是在编辑文本时就会创建的文件，如果程序正常退出，临时文件自动删除，如果意外退出就会保留，文件名为 .filename.swp，（第一次产生的交换文件名为.filename.txt.swp；再次意外退出后，将会产生名为.filename.txt.swo的交换文件；而第三次产生的交换文件则为.filename.txt.swn）

svn泄露


当开发人员使用 SVN 进行版本控制，对站点自动部署。如果配置不当,可能会将.svn文件夹直接部署到线上环境。这就引起了 SVN 泄露漏洞。

需要用到一个工具https://github.com/kost/dvcs-ripper

  • 安装 ：sudo apt-get install perl libio-socket-ssl-perl libdbd-sqlite3-perl libclass-dbi-perl libio-all-lwp-perl
  • 下载：git clone https://github.com/kost/dvcs-ripper
  • 进入目录—-./rip-svn.pl -v -u http://challenge-05b7bb06a8d52597.sandbox.ctfhub.com:10080/.svn/

    svn1.7及以后版本则只在项目根目录生成一个.svn文件夹，里面的pristine文件夹里包含了整个项目的所有文件备份

POST请求

看wp说能扫出index.php和flag.php的，但我没扫出来
直接访问吧
/flag.php——->Just View From 127.0.0.1
那么?url=127.0.0.1/flag.php
可以用file协议读取flag.php 

<?php

error_reporting(0);

if ($_SERVER["REMOTE_ADDR"] != "127.0.0.1") {
    echo "Just View From 127.0.0.1";
    return;
}

$flag=getenv("CTFHUB");
$key = md5($flag);

if (isset($_POST["key"]) && $_POST["key"] == $key) {
    echo $flag;
    exit;
}
?>

没有提交按钮，可以用gopher协议构造post请求
